# Publishing Assets Checklist for Solana Mobile dApp Store

Complete checklist of all assets needed to submit Balance Seekr to the Solana Mobile dApp Store.

## Required Assets

### âœ… 1. App Icon
- **File Name:** `app-icon-512.png`
- **Size:** 512x512 pixels
- **Format:** PNG
- **Requirements:**
  - [ ] Square (1:1 aspect ratio)
  - [ ] No transparency (solid background)
  - [ ] High quality (no pixelation)
  - [ ] Represents your app clearly
  - [ ] Matches your app's visual identity
- **Design Tips:**
  - Use your app logo or a simplified version
  - Ensure it's readable at small sizes
  - Consider rounded corners (will be applied automatically)
  - Use high contrast colors

### âœ… 2. Banner Graphic
- **File Name:** `banner-1200x600.png`
- **Size:** 1200x600 pixels (2:1 aspect ratio)
- **Format:** PNG or JPG
- **Requirements:**
  - [ ] Landscape orientation
  - [ ] High quality (no pixelation)
  - [ ] Represents your app's key features
  - [ ] Includes app name/logo
  - [ ] Visually appealing
- **Design Tips:**
  - Showcase key features (Masterminds, Habits, Journal)
  - Include Solana branding if relevant
  - Use consistent color scheme with your app
  - Keep text minimal and readable

### âœ… 3. Screenshots (Minimum 4, Recommended 6-8)
- **File Names:** `screenshot-1.png`, `screenshot-2.png`, etc.
- **Sizes:** Phone screenshots (typically 1080x1920 or similar)
- **Format:** PNG or JPG
- **Requirements:**
  - [ ] At least 4 screenshots
  - [ ] Show actual app functionality
  - [ ] High quality (no blur)
  - [ ] Portrait orientation (phone screenshots)
  - [ ] No UI overlays or debug elements

#### Recommended Screenshots:

   - Show badges, goals, daily quote

   - Highlights: Solana integration, community features

3. **Mastermind Detail/Group** - [ ]
   - Show group card with background image
   - Highlights: Visual appeal, group info

4. **Mastermind Chat** - [ ]
   - Show chat interface
   - Highlights: Social features, messaging

5. **Habits & Goals** - [ ]
   - Show habit tracking interface
   - Highlights: Productivity features

6. **Journal Entry** - [ ]
   - Show journal writing interface
   - Highlights: Personal growth features

7. **Tasks Page** - [ ]
   - Show task management
   - Highlights: Organization features

8. **Profile/Settings** - [ ]
   - Show user profile with X verification
   - Highlights: User management, verification

**Screenshot Tips:**
- Use a real device or high-quality emulator
- Remove any personal/sensitive data
- Show the best features of your app
- Ensure text is readable
- Use consistent device frame (optional but recommended)

### âœ… 4. Release APK
- **File Name:** `balance-seekr-release.apk`
- **Type:** Signed release build
- **Requirements:**
  - [ ] Built with EAS or locally
  - [ ] Signed with release keystore
  - [ ] Not a debug build
  - [ ] Tested on Android device
  - [ ] All features working correctly
  - [ ] Solana wallet integration tested

## Quality Checklist

Before submitting, verify:

- [ ] All images are high resolution (no pixelation)
- [ ] Colors are consistent across assets
- [ ] Text is readable at all sizes
- [ ] No typos or errors
- [ ] Assets match your app's actual appearance
- [ ] Screenshots show real functionality (not mockups)
- [ ] APK is tested and working
- [ ] All file names are clear and organized
